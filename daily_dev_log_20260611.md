# 🗓️ บันทึกสรุปการปรับปรุงระบบแบบละเอียด (11 มิถุนายน 2026)

วันนี้ทีมพัฒนาได้ดำเนินการแก้ไขปัญหาสำคัญเกี่ยวกับการยกเลิก/ล้างวันกำหนดส่งงาน (Due Date) ซึ่งส่งผลให้แอปพลิเคชันเกิดการแครชและขึ้นหน้าจอสีขาว (White Screen Crash) โดยสรุปรายละเอียดได้ดังนี้ครับ:

---

## 🛠️ 1. แก้ไขหน้าจอขาว (Crash) เมื่อกดยกเลิก/ล้างวันกำหนดส่งงาน (DueDate)
- **ปัญหา**:
    - เมื่อผู้ใช้ทำการล้างวันกำหนดส่งงานในหน้าต่างแก้ไข/เพิ่มงาน (DueDate ถูกล้างเป็นค่าว่าง `""` หรือ `null`) แล้วกดบันทึก หน้าจอแอปพลิเคชันจะดับไปเป็นหน้าขาวทั้งหมด
- **สาเหตุ**:
    - คอลัมน์ `DueDate` ในฐานข้อมูล Supabase ถูกออกแบบเป็นชนิดข้อมูล `TEXT` ซึ่งยอมรับค่าว่าง (`""` หรือ `null`) ได้อย่างถูกต้อง
    - แต่บน Frontend มีโค้ดหลายจุดที่นำค่า `task.DueDate` หรือ `briefing.DueDate` ไปแปลงผ่าน `new Date(...)` และเรียกใช้ฟังก์ชันจัดรูปแบบวันที่ `format()` ของไลบรารี `date-fns` โดยไม่มีการตรวจสอบค่าว่าง
    - การแปลงค่าว่างหรือ `null` ด้วย `new Date(...)` จะส่งคืนวัตถุประเภท `Invalid Date` ซึ่งเมื่อถูกส่งต่อให้กับฟังก์ชัน `format()` จะทำให้เกิด Runtime Error: `RangeError: Invalid time value` และระบบหยุดทำงานทันที (Crash)

- **สิ่งที่ได้ปรับปรุงแก้ไข**:
    - **[Tasks.jsx](file:///d:/จัดสเปค ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/src/views/Tasks.jsx)**:
        - **มุมมองการ์ด (Card View)**: ปรับ Logic การแสดงผลกำหนดส่งวันที่ หาก `DueDate` เป็นค่าว่างจะแสดงเป็น `"ไม่มีกำหนดส่ง"` แทนการฟอร์แมต
        - **มุมมองตาราง (Table View)**: แก้ไขให้แสดงผล `"ไม่มีกำหนดส่ง"` ป้องกันคอลัมน์ในตารางพัง
    - **[Timeline.jsx](file:///d:/จัดสเปค ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/src/views/Timeline.jsx)** (ปฏิทินงานหลัก):
        - ปรับ Logic การแปลงข้อมูล `userTasks` โดยเพิ่ม fallback ให้กับ `endDate` หาก `DueDate` เป็นค่าว่าง ให้ดึงค่า `StartDate` มาเป็นตัวกำหนดขอบเขตไทม์ไลน์แทน เพื่อรักษาการแสดงผลและการนำไปวาดบนปฏิทินให้อยู่รอด
        - แก้ไขส่วนแสดงรายละเอียดวันที่ใน Modal บนหน้าปฏิทินให้แสดงเป็น `"ไม่มีกำหนดส่ง"` หากไม่มีข้อมูล
    - **[StatusTasksModal.jsx](file:///d:/จัดสเปค ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/src/components/StatusTasksModal.jsx)** (หน้ารวมงานตามสถานะ):
        - ปรับเงื่อนไขให้แสดงคำว่า `"ไม่มีกำหนดส่ง"` หากไม่มี DueDate ในกล่องข้อมูลงาน
    - **[Briefing.jsx](file:///d:/จัดสเปค ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/src/views/Briefing.jsx)** (ระบบบรีฟงาน):
        - ปรับการฟอร์แมตวันที่ DueDate ทั้งบนหน้าการ์ดบรีฟ และหน้าตารางบรีฟงาน ให้แสดงผลเป็น `"ไม่มีกำหนดส่ง"` เมื่อเป็นค่าว่าง
    - **[BriefingTimeline.jsx](file:///d:/จัดสเปค ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/src/components/BriefingTimeline.jsx)** (ปฏิทินงานบรีฟ):
        - แก้ไขฟังก์ชัน `getBriefingsForDay` ให้ข้ามการเปรียบเทียบวันของงานบรีฟที่ไม่มี `DueDate` (กรองออกจากการคำนวณวันเวลานั้นๆ) เพื่อไม่ให้คำสั่ง `.setHours()` ขัดข้อง
    - **[DailySummaryModal.jsx](file:///d:/จัดสเปค ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/src/components/DailySummaryModal.jsx)** (ระบบคัดลอกสรุปงานวันนี้):
        - ปรับ Logic การจัดเก็บตัวแปร `tDue` หากไม่มี DueDate ให้ดึง `StartDate` มาแทน ป้องกันเงื่อนไขเปรียบเทียบช่วงวันที่พัง

## 🏷️ 2. ปรับปรุงเวอร์ชันของระบบและ Changelog
- **สิ่งที่ทำ**:
    - อัปเดตข้อมูลไฟล์ `version.json` เพื่อบันทึกการปรับปรุงระบบของวันนี้:
        - ปรับ timestamp `lastUpdated` ล่าสุด
        - เพิ่ม Changelog: `"🛠️ แก้ไขปัญหาหน้าจอขาว (Crash) เมื่อกดยกเลิก/ล้างวันกำหนดส่งงาน (DueDate)"`

## 🧪 3. การตรวจสอบความถูกต้องและการ Push โค้ด (Git Release)
- **สิ่งที่ทำ**:
    - ทำการ Build ตัวโปรแกรมด้วยคำสั่ง `vite build` เพื่อตรวจสอบความเรียบร้อยและป้องกันข้อผิดพลาดในการรันแอปพลิเคชัน
    - บันทึกประวัติและอัปโหลดโค้ดไปยัง GitHub Remote:
        - `git add .` และ commit ภายใต้หัวข้อ `fix: handle empty DueDate safely to prevent white screen crash`
        - ทำการ `git push origin dev` ขึ้นสู่กิ่งพัฒนาระบบ
        - สลับไปยังกิ่งหลัก `main` และ merge โค้ดของวันนี้จาก `dev`
        - ทำการ `git push origin main` เพื่อปล่อยอัปเดตเวอร์ชันแก้ไขอย่างเป็นทางการ
        - สลับกิ่งกลับมาทำงานต่อที่ `dev` เป็นค่าเริ่มต้น
