# 🗓️ บันทึกสรุปการปรับปรุงระบบ (20 กรกฎาคม 2026)

วันนี้ทีมพัฒนาได้ทำการอัปเดตระบบเพิ่มฟังก์ชันการทำงานใหม่และปรับปรุงระบบการแสดงผล ดังนี้ครับ:

เรื่องที่ 1: พัฒนาระบบผู้ช่วย AI สรุปงานและตอบคำถามอัจฉริยะ (WorkLogs AI Chatbot & Mini Chat)
  - ประเภท: เพิ่มฟีเจอร์ใหม่ (New Feature) และวิเคราะห์สถาปัตยกรรม (Architecture & AI Integration)
  - สิ่งที่ทำ:
    - วิเคราะห์และเลือกใช้โมเดลภาษาไทย pathumma-thaillm-qwen3-8b-think-3.0.0 (NECTEC/Qwen3 base) ที่มีความสามารถด้านการคิดวิเคราะห์ (Reasoning/Thinking Mode) สูง
    - ออกแบบสถาปัตยกรรม Direct Browser Client to LLM API (Zero Vercel Server Load) โดยให้ Vercel ทำหน้าที่เป็นเพียง Static Host ไม่ต้องรัน Serverless Functions ประมวลผล AI เพื่อป้องกันการกิน CPU และค่าบริการ Vercel
    - พัฒนาระบบจำกัดอัตราการยิงคำขอ (Client-Side In-Memory Rate Limiter) ควบคุมไม่ให้เกิน 5 คำขอ/วินาที และ 200 คำขอ/นาที ทำงานในความจำเครื่องผู้ใช้ 100% ไร้การส่ง Network Request ค้าง
    - ออกแบบระบบกรองสิทธิ์ข้อมูล (RBAC Pre-Filtering) และตารางจับคู่ชื่อเล่นพนักงาน 10 คน (เหมี่ยว, กีต้า, พอร์มเตอร์, โม, ฟลุ๊ค, บัส, บอส, เติ้ล, แพท, มาย) พร้อมระบบ Fuzzy Matching ดักจับคำพิมพ์ผิดเชิงเสียง (เช่น แพด -> แพท, บอด -> บอส) เพื่อดึงข้อมูลงานใน Supabase มาให้ AI วิเคราะห์ตามสิทธิ์จริง
    - พัฒนาการเชื่อมต่อ DuckDuckGo Instant Answer API เพื่อดึงข้อมูลอินเทอร์เน็ตเมื่อผู้ใช้สอบถามเรื่องทั่วไปนอกเหนือจากงานในระบบ
    - สร้างหน้าแชทหลัก ChatbotWorklog บริหารจัดการห้องแชท (Chat Rooms) จัดเก็บประวัติลงใน localStorage พร้อมแสดงกระบวนการคิด (Thinking) และการ์ดเกร็ดความรู้ (Knowledge Cards)
    - สร้าง Mini Chat ไอคอนแมว Pixel Art Minimal (pixel_cat.png) เป็น Slide Panel สไลด์มาจากด้านขวา ไม่บังปุ่มหรือเนื้อหาบนหน้าจอ พร้อมระบบซ่อนตัวอัตโนมัติเมื่อเข้าสู่หน้า /chatbot
  - ผลลัพธ์: ได้ผู้ช่วย AI ตอบคำถามสรุปงานและค้นหาเว็บเป็นภาษาไทยได้อย่างรวดเร็ว แม่นยำ และปลอดภัยตามสิทธิ์ โดยไม่กระทบต่อทรัพยากร Vercel 100%
  - ไฟล์ที่เกี่ยวข้อง: [thaiLlmService.js](file:///d:/จัดสเปค%20ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/src/services/thaiLlmService.js), [ChatbotWorklog.jsx](file:///d:/จัดสเปค%20ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/src/views/ChatbotWorklog.jsx), [ChatbotWorklog.css](file:///d:/จัดสเปค%20ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/src/views/ChatbotWorklog.css), [MiniChatBot.jsx](file:///d:/จัดสเปค%20ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/src/components/MiniChatBot.jsx), [Layout.jsx](file:///d:/จัดสเปค%20ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/src/components/Layout.jsx), [App.jsx](file:///d:/จัดสเปค%20ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/src/App.jsx)

เรื่องที่ 2: แก้ไขปัญหา Context Token Exceeded ของโมเดล ThaiLLM (API Error 400 Bad Request Fix)
  - ประเภท: แก้ไขบั๊ก (Bug Fix) และปรับปรุงประสิทธิภาพ (Performance Optimization)
  - สิ่งที่ทำ:
    - วิเคราะห์สาเหตุข้อผิดพลาด 400 Bad Request เกิดจากโมเดลมี Context Limit รวมที่ 16,384 Tokens แต่ระบบส่ง Prompt ข้อมูลงาน 15,333 Tokens บวกกับ max_tokens สำหรับคำตอบ 8,192 Tokens รวมเป็น 23,525 Tokens เกินโควต้า
    - ปรับปรุงการตั้งค่า max_tokens สำหรับคำตอบลงมาอยู่ที่ 3,072 Tokens (เพียงพอสำหรับคำตอบยาว 2,000-3,000 คำ)
    - ปรับกระชับข้อมูลงานใน System Prompt โดยลดรายการงานล่าสุดเหลือ 15 งาน และบรีฟเหลือ 10 รายการ พร้อมจำกัดความยาว System Prompt ไม่ให้เกิน 8,000 ตัวอักษร (~2,500 Tokens) และส่ง History ย้อนหลัง 5 ข้อความล่าสุด
    - เพิ่มระบบ Proxy การยิง API ผ่าน vite.config.js (server.proxy) ในช่วง Local Dev และผ่าน vercel.json (rewrites) ในช่วง Production เพื่อแก้ไขปัญหา CORS
  - ผลลัพธ์: แก้ไขปัญหา API Error 400 Bad Request ได้อย่างเด็ดขาด ระบบสามารถประมวลผลคำตอบได้ราบรื่น รัน Build ผ่าน 100%
  - ไฟล์ที่เกี่ยวข้อง: [thaiLlmService.js](file:///d:/จัดสเปค%20ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/src/services/thaiLlmService.js), [vite.config.js](file:///d:/จัดสเปค%20ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/vite.config.js), [vercel.json](file:///d:/จัดสเปค%20ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/vercel.json)

เรื่องที่ 3: แก้ไขปัญหาระบบเปลี่ยนสถานะบรีฟงานไม่อัปเดตอัตโนมัติ (Briefing Auto-Status Update Fix)
  - ประเภท: แก้ไขบั๊ก (Bug Fix)
  - สิ่งที่ทำ:
    - ตรวจสอบลอจิกในฟังก์ชัน handleSaveResponse ของหน้าต่างบรีฟงาน พบว่ามีเงื่อนไข isFirstRecord จำกัดการเปลี่ยนสถานะไว้เฉพาะการส่งงานครั้งแรก ทำให้เมื่อผู้รับมอบหมายแนบงานแก้ไขในครั้งถัดๆ ไป สถานะไม่ยอมเปลี่ยน
    - ปรับปรุงเงื่อนไขใหม่โดยลบ isFirstRecord ออก บังคับให้ระบบเปลี่ยนสถานะเป็น "รอตรวจ" อัตโนมัติทุกครั้งที่มีการแนบไฟล์รูปภาพ ลิงก์ URL หรือโน้ตข้อความผลงาน (ยกเว้นสถานะที่เป็น 'เสร็จสิ้น', 'ยกเลิกงาน' หรือ 'รอตรวจ' อยู่แล้ว)
  - ผลลัพธ์: ผู้รับมอบหมายส่งงานหรือแนบไฟล์แก้ไขกี่ครั้ง สถานะจะถูกเปลี่ยนเป็น "รอตรวจ" เพื่อแจ้งหัวหน้างานทราบทันทีอย่างถูกต้องแม่นยำ
  - ไฟล์ที่เกี่ยวข้อง: [BriefingModal.jsx](file:///d:/จัดสเปค%20ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/src/components/BriefingModal.jsx)

เรื่องที่ 4: แก้ไขปัญหาการแสดงผลลิงก์แนบที่มีความยาวมากล้นขอบจอ (Briefing Attached Link UI Fix)

  - สิ่งที่ทำ:
    - แก้ไขการ rendering ลิงก์แนบ (URL1, URL2) ในหน้าต่างบรีฟงาน เพื่อแก้ปัญหาลิงก์ความยาวสูง (เช่น Google Drive, SharePoint, GitHub) ล้นขอบการ์ดหรือดัน Layout เบี้ยว
    - เพิ่มคลาส break-all, min-w-0, flex-1 ให้ข้อความ URL เพื่อให้สามารถตัดคำและขึ้นบรรทัดใหม่ได้แบบอัตโนมัติ พร้อมใส่ shrink-0 ให้ไอคอนลิงก์ภายนอก
    - ปรับแต่งดีไซน์กรอบการ์ดลิงก์แนบใหม่ด้วย bg-blue-50/90 border border-blue-100 p-2.5 rounded-xl พร้อมใส่ tooltip title ให้ดูสวยงามและเป็นระเบียบ
  - ผลลัพธ์: ลิงก์แนบทุกความยาวแสดงผลได้เป็นระเบียบ สวยงาม ไม่ล้นขอบจอ และสามารถกดเปิดลิงก์ได้ตามปกติ
  - ไฟล์ที่เกี่ยวข้อง: [BriefingModal.jsx](file:///d:/จัดสเปค%20ยื่นข้อเสนอ/ระบบลงบันทึกงาน/frontend/src/components/BriefingModal.jsx)

เรื่องที่ 5: ยกระดับผู้ช่วยเป็น CatLog AI
  - ปรับชื่อและหน้าจอแชทเป็น CatLog AI พร้อมไอคอนแมวพื้นหลังโปร่งใส และปรับหน้าต่างแจ้งอัปเดตให้อ่านง่ายขึ้น
  - เพิ่มค้นหาข้อมูลภายนอกพร้อมลิงก์แหล่งอ้างอิง และรองรับวางข้อความยาวเพื่อสรุปประเด็น/สิ่งที่ต้องทำต่อ
  - แยกข้อมูลเว็บออกจากข้อมูล WorkLogs เพื่อไม่ให้ตัวเลขภายในระบบคลาดเคลื่อน
  - ผลลัพธ์: ใช้ถามงาน ค้นข้อมูล และสรุปข้อความได้ในที่เดียว

เรื่องที่ 6: ทำให้คำตอบข้อมูลพนักงานและสถิติงานแม่นยำขึ้น
  - ชื่อเล่นพนักงานจะถูกจับคู่เป็นรหัสพนักงานก่อนค้นหา ลดปัญหาชื่อหรือคำนำหน้าชื่อไม่ตรง
  - สรุป Dashboard และ My Team คำนวณจากฐานข้อมูลตามสิทธิ์ แทนการให้ AI เดาตัวเลข
  - แก้การเชื่อมต่อ AI Local และเพิ่มทางสำรองเมื่อ session ของ AI ใช้งานไม่ได้
  - ผลลัพธ์: คำตอบเรื่องงาน บรีฟ และคะแนนสอดคล้องกับหน้าระบบมากขึ้น

เรื่องที่ 7: ปรับ CatLog AI ให้ตอบตรงชนิดข้อมูลและทดสอบกับข้อมูลจริง
  - แยกคำถามเป็น Dashboard, Briefing และ My Team เพื่อไม่ให้นับผิดตาราง และตอบตัวเลขจาก Supabase โดยตรง
  - จับคู่ชื่อเล่น/ชื่อจริงกับรหัสพนักงานตามสิทธิ์ รวมถึงแบ่งโหลดผลบรีฟเป็นชุดย่อยเพื่อแก้คำถามคะแนนล้มเมื่อข้อมูลมาก
  - ปรับค้นเว็บแบบ Bing-first ให้จัดอันดับความเกี่ยวข้อง ตัดลิงก์เปลี่ยนทาง และแยกการสรุปข้อความยาวออกจากข้อมูล WorkLogs
  - ทดสอบอัตโนมัติผ่าน 19 รายการ, production build ผ่าน และทดสอบจริงพบงานแพท 660 งาน, บรีฟทั้งหมด 537 งาน และคะแนนแพทเดือน 07/2026 เท่ากับ 183 คะแนนตรงกับหน้าระบบ

เรื่องที่ 8: เพิ่มโหมดสรุปข่าวเหตุการณ์
  - ใช้ Bing News RSS ดึงหัวข่าวจริง วันที่ สำนักข่าว และบทความต้นทาง พร้อมกรองหน้าหมวดข่าวเก่า
  - แยกค้นข่าวฆาตกรรม/ปล้นขโมย/เพลิงไหม้ และสรุปสถานที่กับยอดเสียชีวิตหรือบาดเจ็บเฉพาะที่แหล่งข่าวระบุ
  - ชุดทดสอบรวมผ่าน 21 รายการ และคำถามข่าวจริงได้ 6 ข่าวพร้อมวันที่และแหล่งอ้างอิงภายในประมาณ 3 วินาที

เรื่องที่ 9: แก้ปุ่มย่อ Mini Chat มองไม่เห็น
  - เปลี่ยนวงกลมโปร่งใสที่มีไอคอนสีขาวเป็นปุ่มไล่สีม่วงพร้อมลูกศรปิดแผง
  - เพิ่ม tooltip และ aria-label ให้ทราบชัดว่าเป็นปุ่มเปิดหรือปิด CatLog AI

เรื่องที่ 10: ผูกคำถามข้อมูลตัวเองกับบัญชีและเพิ่มระดับสิทธิ์
  - คำว่า ฉัน/ผม/ตัวเอง ใช้ User ID จาก session และไม่ใช้ชื่อพนักงานที่ค้างจากตัวกรอง Dashboard
  - Staff ดูได้เฉพาะตัวเอง, Head ดูได้เฉพาะตนเองและคนในแผนก, Admin ดูข้อมูลพนักงานได้ทั้งหมด
  - เพิ่มการ์ดระดับสิทธิ์ในหน้าแชท และทดสอบรวมผ่าน 23 รายการ

เรื่องที่ 11: ทำสรุปข่าวให้อ่านง่ายและเพิ่มคู่มือใช้งาน
  - รวมบทความที่รายงานเหตุการณ์เดียวกันเป็นสรุปเดียว พร้อมยอดล่าสุดและสำนักข่าวที่ยืนยัน
  - ย้ายลิงก์อ้างอิงทั้งหมดไปส่วนแหล่งข้อมูลท้ายคำตอบ ลดข่าวซ้ำและข้อความยาว
  - เพิ่มคู่มือ 4 โหมด: ข้อมูลในระบบ, ค้นเว็บ/ข่าว, สรุปข้อความ และสูตรตั้งคำถาม พร้อมปุ่มตัวอย่าง
  - ชุดทดสอบรวมผ่าน 24 รายการ และ production build ผ่าน

เรื่องที่ 12: ทำให้ปุ่ม CatLog AI สังเกตและเข้าใจง่ายขึ้น
