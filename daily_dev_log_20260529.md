# 🗓️ บันทึกสรุปการปรับปรุงระบบแบบละเอียด (29 พฤษภาคม 2026)

วันนี้ทีมพัฒนาได้ปรับปรุงและขยายขีดจำกัดระบบแนบรูปภาพให้มีความคมชัดสูงขึ้นตามความต้องการของผู้ใช้งาน พร้อมทั้งปรับปรุงเวอร์ชันของระบบ โดยสรุปรายละเอียดและแนวทางการดำเนินงานได้ดังนี้ครับ:

---

## 📸 1. ปรับปรุงระบบแนบรูปภาพความละเอียดสูง (2MB Limit & Gradual Compression)
- ทำอะไรไปบ้าง:
    - ขยายขีดจำกัดขนาดไฟล์รูปภาพแนบสูงสุดในการรายงานผลงานหลัก (Tasks) และในระบบบรีฟงาน (Briefings - ทั้งภาพอ้างอิง ภาพผลงาน และภาพคอมเมนต์การตรวจงาน) จากขีดจำกัดเดิม (~450KB หรือ 600,000 ตัวอักษร base64) เพิ่มขึ้นเป็นสูงสุดไม่เกิน 2MB (2,097,152 Bytes)
    - เปลี่ยนตรรกะการบีบอัดรูปภาพใหม่ทั้งหมด โดยใช้ Gradual Compression Loop (ลูปบีบอัดค่อยเป็นค่อยไป) แทนระบบเดิมที่เป็นแบบ Recursion:
        - ความชัดระดับสูงสุดเริ่มต้น: เริ่มต้นด้วยขนาดภาพกว้าง/สูงสูงสุดที่ 2560px (รักษาอัตราส่วนภาพ) และระดับคุณภาพของ WebP สูงสุดที่ 0.9 เพื่อถนอมรายละเอียดและความคมชัดของรูปภาพให้ออกมาดีที่สุด (กรณีรูปภาพต้นฉบับมีขนาดเล็กกว่าจะไม่ถูกดึงให้ขยายขึ้น)
        - การลดขนาดไฟล์อัจฉริยะ (ค่อยๆ ลด): หากขนาดไฟล์หลังบีบอัดยังคงเกินเกณฑ์ 2MB ระบบจะค่อยๆ ปรับลดระดับความคมชัดลงทีละ 0.05 จนถึงระดับต่ำสุดที่ยอมรับได้คือ 0.3
        - การย่อขนาดมิติภาพ: หากปรับระดับคุณภาพต่ำถึง 0.3 แล้วไฟล์ยังมีขนาดเกิน 2MB ระบบจะย่อขนาดความกว้าง/สูงของรูปภาพลงครั้งละ 20% (คูณ 0.8) และปรับค่าระดับคุณภาพกลับมาที่ 0.8 เพื่อเริ่มบีบอัดใหม่อีกครั้ง จนกว่าจะได้ขนาดไฟล์ไม่เกิน 2MB
        - ความปลอดภัยของระบบ: การนำลูป while มาเขียนควบคุมตรรกะนี้แทนการเรียกฟังก์ชันซ้อนกัน (Recursion) ช่วยรับประกันว่าการประมวลผลบนเครื่องเบราว์เซอร์ของผู้ใช้จะไม่มีทางเกิดปัญหา Stack Overflow (Maximum call stack size exceeded) 100%
- ผลลัพธ์:
    - ผู้ใช้สามารถอัปโหลดหรือวางรูปภาพแนบในระบบได้ด้วยขนาดไฟล์ที่สูงสุดถึง 2MB โดยที่รูปภาพยังคงรักษาความละเอียด คมชัด และไม่มีการแตกของภาพสูงกว่าเดิมอย่างเห็นได้ชัด

## 🏷️ 2. ปรับปรุงเวอร์ชันของระบบและการอัปเดต (Version & Changelog Update)
- ทำอะไรไปบ้าง:
    - ปรับปรุงเลขรุ่นเวอร์ชันในไฟล์ package.json จากเวอร์ชัน 0.0.1 ขึ้นเป็น 0.0.2
    - เขียนบันทึกการปรับปรุงระบบ (Changelog) ล่าสุดระบุถึงการอัปเกรดระบบ Limit รูปภาพ 2MB และ Gradual Compression แทรกไว้ที่บนสุดของไฟล์ changelog.json เพื่ออัปเดตข้อมูลการอัปเดตระบบแก่ผู้ใช้งานคนอื่นๆ

## 🧪 3. การทดสอบและการส่งมอบงาน (Verification & Git Operations)
- ทำอะไรไปบ้าง:
    - ทำการ Build แอปพลิเคชันด้วยคำสั่ง npm run build ตรวจสอบความถูกต้องของไวยากรณ์ (Syntax) และการคอมไพล์ของระบบเรียบร้อยแล้ว โดยไม่มีข้อผิดพลาดใดๆ
    - ทำการบันทึกประวัติโค้ดที่แก้ไขและดำเนินกระบวนการ Git ทั้งหมด:
        - สั่ง git commit การเปลี่ยนแปลงเข้าสู่สาขาพัฒนาหลัก (dev)
        - ทำการ git push origin dev ขึ้นสู่ระบบ GitHub
        - สลับไปยังสาขาหลัก (main) ด้วย git checkout main
        - ดำเนินการ Merge โค้ดพัฒนาล่าสุดด้วย git merge dev
        - ทำการ git push origin main เพื่อเผยแพร่เวอร์ชันใหม่ขึ้นสู่สภาพแวดล้อมหลัก
        - สลับสาขากลับมายัง dev เพื่อเปิดรับการพัฒนาในขั้นตอนต่อไป
